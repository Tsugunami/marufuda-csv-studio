# 丸札CSVスタジオ (marufuda-csv-studio)

工事現場向け **丸札（機器行き先ラベル）** 作成支援ツール。

A-ONE ラベル屋さん™ の **差し込み印刷機能** で読み込める CSV を、Excel マクロを使わずにモダンなデスクトップアプリで作成する。

> 従来の `KintoneJS_Single/丸札マクロ` (VBA) を Windows ネイティブ化・環境非依存化した後継ツール。

---

## ✨ 特長

- **Excel 不要** — Tauri + React で動く軽量デスクトップアプリ
- **ブロック構造エディタ** — `列×行` のブロック格子と「1ラベルあたりの行数」を UI で柔軟設定
- **ラベルサイズ（縦横 mm）指定** — 全体ビューにアスペクト比が反映され、実際のラベル見た目をイメージしやすい
- **「～」デリミタ位置の制御** — 偶数行時に「自分側 / 相手側」のどちらに寄せるかをラベルごとに個別設定可能（奇数行は中央固定）
- **反転コピー機能** — 自機器 ↔ 相手機器 の表示入れ替えをワンクリックで隣接ラベルへ転記（デリミタ位置も自動反転、ラベル個別の寄せ設定に対応）
- **全体ビュー** — ブロック全体をズーム可能なキャンバスで俯瞰。ラベルのmm比率と出力用フォントサイズで表示し、クリック選択・左上のサイズ表示・ドラッグ移動に対応
- **全体ビュー直接編集** — ダブルクリックで編集中のセルにインライン入力、デリミタ設定も画面上から切替可能
- **複数選択・一括操作** — Ctrl+クリックで個別選択、Shift+クリックで範囲選択、Del で一括クリア、Ctrl+Z でUndo、**Ctrl+C/V でコピー＆ペースト**（反転コピー状態の場合は反転テキストが貼付される）
- **プリセットラベル（定型文／カテゴリタブ管理）** — よく使う行テキストを保存し、全体ビューの上部バーからワンクリックでカーソル位置に貼付。
  カテゴリタブで分類管理でき、⇅並び替えモードでラベルの並び替え・カテゴリ移動・カテゴリ名の変更・削除が可能
- **プロジェクト保存・再開** — 入力状態・設定・プリセットをJSONファイルに保存・読み込み可能
- **残りラベルシート再利用** — 使用済み位置をアプリ共通で保存し、同じ用紙レイアウトの未使用ラベルだけを次の作業で再利用可能
- **CSV/Excel読込** — CSV または xlsx を読み込んで編集再開可能（Shift-JIS/UTF-8自動判定、プリセット自動マッチング）
- **ファイル名指定** — レイアウト設定のファイル名欄がCSV出力時のデフォルト名になる
- **CSV 出力** — Shift-JIS / UTF-8(BOM) / UTF-8 を切替可能、ラベル屋さん差し込み印刷にそのまま利用可能（ヘッダ行なし、1行目から直接データ）
- **Excel (xlsx) 出力** — 同じデータを Excel 形式でも出力可能
- **プリセット** — よく使うレイアウトを保存・削除・上書き保存・再利用（デフォルト7種同梱）
  - プリセット切替時は入力データがクリアされることを確認するモーダル表示
  - 削除時も確認モーダル表示

## 🧩 ユースケース

工事現場で丸札（機器の行き先を示す札）を作成する際、1枚のラベルに「自機器〜相手機器」の情報を表示する。相手側にも同じラベルを貼るため、表示を入れ替えたラベルをもう1枚作る必要がある。

本ツールでは:

1. ブロック格子の全体ビューから対象ラベルを選ぶ
2. 右側のエディタで行データを入力 or 全体ビューでダブルクリックして直接入力
3. **反転ボタン** で「～」を境に入れ替えたデータを隣ラベルへ自動転記
4. 全ラベル入力後、**CSV出力** で1ラベル=1行のフラットな CSV を生成
5. 出力した CSV を **ラベル屋さん™ ソフト** の差し込み印刷で読み込み、ラベル用紙に印刷

### 部分使用のユースケース

「初回はラベル上半分のみ使用し、次回以降で下半分を使用」のような運用にも対応。1文字も入力されていないラベルはデリミタ「～」も含めて完全に空白となり、CSV出力でも空行として出力される。

## 📐 ブロック構造と「～」挿入ロジック

### ブロック構造

```
シート全体 = blockCols × blockRows の格子
各セル = 1ラベル（itemsPerLabel 行のデータを保持）

例: blockCols=5, blockRows=8, itemsPerLabel=7
→ 5×8 = 40 ラベル、各ラベル 7 行
```

### 「～」の挿入位置

| itemsPerLabel | delimiterAlign | ～の行 index (0始まり) |
|---|---|---|
| 奇数 (例: 7) | (強制 center) | `floor(n/2)` = 3 |
| 偶数 (例: 6) | `self` (自分側=上に寄せる) | `n/2 - 1` = 2 |
| 偶数 (例: 6) | `partner` (相手側=下に寄せる) | `n/2` = 3 |

#### 6行の例

**`delimiterAlign = 'partner'`（～を相手側に寄せる）**
```
あああ      ← 0 自分
いいい      ← 1 自分
ううう      ← 2 自分
～          ← 3
えええ      ← 4 相手
おおお      ← 5 相手
```

**`delimiterAlign = 'self'`（～を自分側に寄せる）**
```
あああ      ← 0 自分
いいい      ← 1 自分
～          ← 2
ううう      ← 3 相手
えええ      ← 4 相手
おおお      ← 5 相手
```

### デリミタ補完ルール

- **使用中ラベル**（1文字でも入力あり）: デリミタ行に「～」を自動表示
- **未使用ラベル**（1文字も入力なし）: デリミタ行も空白

これにより「初回は上半分のみ使用、次回は下半分を使用」のような部分使用が可能。

## 🔄 反転コピー

選択ラベルの「～」を境に前後を入れ替え、隣接ラベル（右/左/上/下）へ値のみ転記する。

```

あああ           えええ
いいい           おおお
ううう           (空)
～               (空)
えええ           ううう
おおお           いいい
                 あああ
                 ～
```

## 🏷️ プリセットラベル（定型文／カテゴリ管理）

全体ビュー上部に配置されたプリセットラベルバーで、よく使うラベルテキストを管理できます。

### カテゴリタブ

- プリセットラベルはカテゴリ（タブ）で分類されます
- デフォルトで「基本」カテゴリが用意されています
- タブをクリックしてカテゴリを切り替えられます
- `⇅` ボタンで **並び替えモード** に切り替えると、カテゴリタブの並び替え・リネーム・削除が可能

### 並び替えモード（⇅ボタン）

並び替えモードを有効にすると、通常のラベル貼付機能は無効になり、代わりに以下のD&D操作が可能になります。

| 操作 | 方法 |
|------|------|
| **ラベルの並び替え** | ラベルをドラッグし、他のラベルの左/右にドロップ（青い罫線が挿入位置を示す） |
| **ラベルのカテゴリ移動** | ラベルを別のカテゴリタブにドロップ |
| **カテゴリタブの並び替え** | タブ自体をドラッグして順序変更 |
| **カテゴリ名の変更** | タブをダブルクリック |
| **カテゴリの削除** | ✕ボタン（所属ラベルも全削除） |
| **カテゴリの追加** | ＋ボタン |
| **ラベルの削除** | 各ラベルの✕ボタン |

> **重要**: WebView2（Tauri on Windows）の制約により、`tauri.conf.json` で `dragDropEnabled: false` に設定しています。これによりHTML5 D&Dが正しく動作します。外部ファイルのD&Dは無効化されます。

### 通常モード（並び替えモードOFF時）

- 各ラベルをクリックすると、全体ビューで編集中のセルにテキストが貼付されます
- 編集モードでない場合、ラベルはクリック不可になります
- ラベルの追加は、カテゴリ名横の入力欄にカンマ区切りで入力し「追加」ボタンをクリック

### Ctrl+C / Ctrl+V によるコピー＆ペースト

- **Ctrl+C**: 選択セルの内容を通常コピー（クリップボードモード: copy）
- **Ctrl+V**: クリップボードの内容を選択セルに貼り付け
- **反転コピーとの連携**: 反転コピーボタンでコピーした場合、`clipboardMode` が `"reverse"` になります。その状態で Ctrl+V を押すと、**反転状態で貼り付け**されます。
- これらのショートカットは全体ビュー（Canvas）にフォーカスがある場合のみ有効です。入力フィールド内ではブラウザ標準の挙動になります。

## ⚠️ 注意事項・既知の制約

### Windows WebView2 D&Dの制限
- Tauri v2 on Windows では、`dragDropEnabled` を `false` に設定しないとHTML5 D&Dが動作しません（Issue [#15138](https://github.com/tauri-apps/tauri/issues/15138)）
- これにより、外部ファイルのドロップ（OS→アプリ）も無効化されますが、本アプリでは「CSV読込」ボタン経由でファイルを開く方式のため実用上の問題はありません
- 内部的なラベルのD&Dによる並び替えは正常に動作します

### MSIインストーラビルド
- 製品名に日本語が含まれているため、WiX Toolset で MSI インストーラの生成に失敗する場合があります
- 必要に応じて `productName` を英数字のみに変更してください

### ラベルプリセットの永続化
- プリセットラベルとカテゴリ構成は、プロジェクトJSONファイルへの保存・読み込み、または自動保存（30秒おき）で永続化されます
- 手動での明示的な保存を行わない場合、アプリ終了時の最終状態が次回起動時に自動復元されます

### 終了時の自動保存
- 編集内容は変更後500msで自動保存され、30秒ごとにも保存されます
- 終了イベントをアプリ側で抑止しないため、✕ボタンおよび `Alt+F4` はWindows標準の終了操作として即時に閉じられます

### ラベル文字数制限
- ラベルサイズと行数から自動計算された最大文字数（実測に基づく+1文字の余裕を含む）を超える入力は、エラー表示されます
- 詳細は「ラベル文字数制限」セクションを参照

### 残りラベルシートの再利用
- ヘッダーの「新規作成」から、白紙シートまたは保存済みの残りラベルシートを選択できます
- 保存履歴からも残りシートを作成できます。履歴に本文があるセルを使用済みとして判定し、空きセルのみを再利用します
- 白紙シートに任意の名前を付けると、使用済み位置を記録する再利用用シートとして登録されます
- 「印刷済みに更新」を押すと、入力済みセルを使用済みとして登録します。CSV出力だけでは使用済みにならないため、未印刷のデータを誤ってロックしません
- 再利用中は使用済みセルがグレーアウトされ、選択・編集・コピー先にはできません

## 📄 CSV 入出力仕様

### CSV 出力

- 1ラベル = 1行（ヘッダ行なし、1行目から直接データ）
- 列順: ラベルの行0, 行1, ..., 行N
- エスケープ: RFC 4180 準拠（カンマ / `"` / 改行を含む場合は `"..."` で囲む）
- 文字コード: **Shift-JIS**（既定・ラベル屋さん互換）/ UTF-8 BOM 付き / UTF-8
- 未使用ラベルは空行として出力

### CSV 読込

- Shift-JIS / UTF-8(BOM) / UTF-8 を自動判定して読み込み
- ヘッダ行（「項目1..項目N」）を自動検出してスキップ
- CSVのラベル数・行数が現在のレイアウトと一致しない場合、プリセット一覧から一致するものを検索
- 一致するプリセットが見つかった場合、モーダルで切り替え確認
- 読み込み時に既存データがある場合、上書き確認モーダルを表示
- 読み込んだファイル名をファイル名欄に自動反映

### インストール

```bash
npm install
```

### 開発起動（HMR付き）

```bash
npm run tauri dev
```

### 本番ビルド（配布用 exe）

```bash
# exe 単体ビルド（tauri.conf.json の bundle.targets が空のためインストーラは生成されない）
npm run tauri build

# 出力先: src-tauri/target/release/marufuda-csv-studio.exe
```

### 配布方法

現時点では **exe 単体配布** が最適です。

1. `npm run tauri build` でビルド
2. `src-tauri/target/release/marufuda-csv-studio.exe` を配布
3. ユーザーは好きな場所に置いてダブルクリックで起動
4. アップデート時は新しい exe で上書きするだけ

> **型チェック・フロントエンド単体ビルド**

> ```

> **特别な指示がない限り、インストーラービルドは不要です。**

## 🔧 トラブルシューティング

### `link.exe` not found
Visual Studio Build Tools の C++ ワークロードが未インストールです。セットアップ手順の Build Tools インストールを実行してください。

### Tauri パッケージのバージョン不整合エラー
`src-tauri/Cargo.toml` の Tauri crate バージョンと `package.json` の `@tauri-apps/*` npm パッケージの major/minor を揃えてください。

### MSI バンドル作成失敗（WiX `light.exe` エラー）
WiX Tools の問題です。実行ファイル（exe）単体は正常にビルドされているため、`src-tauri/target/release/marufuda-csv-studio.exe` を直接実行できます。

### アプリ起動中にビルドできない
起動中の exe がロックしているためビルドエラーになります。アプリを終了してから再ビルドしてください。
